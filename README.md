# SYwork 工时记录系统

## 项目简介

SYwork 是一款功能强大的个人工时记录微信小程序，旨在帮助用户高效地规划、跟踪和管理工作时间。通过简洁直观的界面和丰富的功能，用户可以轻松制定工作计划、安排日常排班、统计工时数据，并进行个人数据管理和多设备同步。

## 功能特点

### 📋 计划管理
- 创建和管理班次模板
- 班次类型选择和自定义颜色设置
- 渐变颜色条选择器，支持自定义班次颜色
- 精确工时设置（支持小时+分钟）
- 班次开始和结束时间设置
- 班次模板编辑和删除功能

### 📅 排班系统
- 制定周/月度工作排班表
- 可视化展示工作时间安排
- 支持底部滚动班次选择器组件
- 排班删除功能，在班次匹配时显示删除按钮
- 排班自定义标签功能，可为排班添加个性化标签
- 周视图图片管理功能，支持添加、查看和删除图片
- 图片自动命名，采用年月周格式
- "回到今天"功能，快速切换到当前日期
- 微信分享功能，支持分享月度排班给好友查看

### 📊 数据统计
- 图表化展示工时统计数据（折线图/柱状图）
- 工时热力图，直观展示工时分布情况
- 累计工时统计，查看全部历史总工时数据
- 班次统计饼图，一目了然查看班次占比
- 分析工作效率和时间分配
- 导出统计报表（CSV格式）
- 本周和当月工时统计功能
- 自定义标准工时设置
- 快速选择日期范围功能（支持周/月/年快速选择）
- 图表中添加标准工时参考线，方便与实际工时对比

### 👤 个人中心
- 个人信息管理（头像、用户名）
- 表情头像功能，支持表情选择和情绪分类
- 用户管理功能（修改昵称、修改密码、切换账号、删除账户）
- 密码加密功能，保护账号安全
- 记住密码和自动恢复数据选项
- 数据导出和导入（JSON和ZIP格式）
- 云备份和同步功能，支持多设备数据同步
- 微信客服功能（联系乐乐）
- 捐赠支持功能
- 更新日志展示
- 缓存清理功能

### 📚 文档中心
- 详细的使用说明和功能介绍
- 快速导航栏，支持滚动固定
- 分类展示各项功能的使用指南

## 技术栈

- **前端框架**：微信小程序原生（TypeScript）
- **类型系统**：TypeScript ^5.5.4，严格模式开启
- **类型定义**：miniprogram-api-typings ^3.12.3 | types/ 目录自定义类型声明
- **核心依赖**：crypto-js ^4.2.0（AES加密）
- **代码规范**：ESLint ^10.2.1（flat config 格式，@typescript-eslint 插件）
- **构建工具**：version-manager.js（版本同步）、sync_changelog.js（更新日志同步）
- **数据存储**：微信本地存储 + 云开发数据库
- **数据同步**：云备份（云函数）
- **第三方库**：vendor/jszip.min.js（ZIP压缩）
- **推荐库**：dayjs（日期处理）、wx-charts / echarts-for-weixin（图表）

## 平台适配

- **HarmonyOS**：支持HarmonyOS平台，适配横屏竖屏和平板设备
- **响应式设计**：针对不同屏幕尺寸和设备类型进行优化

## 快速开始

### 环境要求
- 微信开发者工具
- Node.js 14.0+

### 安装步骤
1. 克隆代码仓库
2. 安装依赖：`npm install`
3. 复制 `env.js.example` 为 `env.js`，填入云环境ID和AppID
4. 在微信开发者工具中打开项目
5. 编译并运行

### 开发流程
1. **开发环境搭建**
   - 安装微信开发者工具
   - 克隆代码仓库
   - 安装依赖：`npm install`
   - 配置 `env.js`（从 `env.js.example` 复制）

2. **开发与调试**
   - 使用微信开发者工具进行开发和调试
   - 代码提交前运行类型检查：`npm run tsc:check`
   - 代码提交前运行代码规范检查：`npm run lint`

3. **代码提交**
   - 提交信息格式：`[模块名]: 变更描述`
   - 版本号使用语义化版本（如v1.0.0）

4. **构建与发布**
   - 构建命令：`npm run build`（tsc检查 + 版本同步 + 更新日志同步）
   - 使用微信开发者工具上传代码
   - 提交审核并发布

### 可用脚本

| 命令 | 说明 |
|------|------|
| `npm run tsc:check` | TypeScript 类型检查（tsc --noEmit） |
| `npm run lint` | ESLint 代码规范检查 |
| `npm run lint:fix` | ESLint 自动修复 |
| `npm run update-version` | 同步版本号到各配置文件 |
| `npm run sync-changelog` | 同步 CHANGELOG.md → utils/changelog.ts |
| `npm run build` | 完整构建（tsc检查 + 版本同步 + 日志同步） |
| `npm run dev` | 开发构建（版本同步 + 日志同步，跳过tsc检查） |

## 数据管理

### 数据导出
- **JSON格式**：完整的数据结构，适合备份和迁移
- **ZIP格式**：包含图片和数据的完整备份
- **导出内容**：
  - 自定义班次模板数据（包括排列顺序）
  - 排班数据
  - 统计数据（根据排班实时计算）
  - 图片数据（仅ZIP格式）

### 数据导入
- 支持从JSON和ZIP文件导入数据
- 导入时会将各个数据恢复至对应板块
- 统计数据会根据导入的排班数据实时更新

### 云备份
- 支持云备份和同步功能，实现多设备数据同步
- 自动管理备份文件，只保留最新版本
- 多设备同步时以最新备份为准，确保数据一致性
- 备份系统版本管理，确保前后端兼容性

## 数据安全

- 用户敏感数据采用 AES 加密存储，禁止明文
- 密码加密功能，保护账号安全
- 密钥配置通过 `env.js` 文件管理（不入库）
- `env.js` 缺失时以空字符串兜底并打印警告
- 避免 eval 等危险函数，过滤用户输入防 XSS

## 注意事项

- 清除小程序缓存会导致数据丢失
- 建议定期导出重要数据作为备份
- 通过云备份功能可以实现多设备数据同步
- 修改云函数后需单独安装生产依赖，禁止上传 node_modules
