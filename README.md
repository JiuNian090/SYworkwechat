# SYwork 工时记录系统

## 项目简介

SYwork 是一款功能强大的个人工时记录微信小程序，旨在帮助用户高效地规划、跟踪和管理工作时间。通过简洁直观的界面和丰富的功能，用户可以轻松制定工作计划、安排日常排班、统计工时数据，并进行个人数据管理和多设备同步。

## 核心功能

### 📋 计划页面 (Plan)
- 创建和管理每日工作任务
- 设置任务优先级和预计工时
- 查看任务完成状态
- 班次模板编辑、复制和批量删除功能
- 模板弹窗优化，支持滚动和触摸事件处理

### 📅 排班页面 (Schedule)
- 制定周/月度工作排班表
- 可视化展示工作时间安排
- 支持底部滚动班次选择器组件
- 排班删除功能，在班次匹配时显示删除按钮
- 周视图图片管理功能，支持添加、查看和删除图片
- "回到今天"功能，快速切换到当前日期

### 📊 统计页面 (Statistics)
- 图表化展示工时统计数据
- 分析工作效率和时间分配
- 导出统计报表（CSV格式）
- 本周和当月工时统计功能
- 自定义标准工时设置
- 快速选择日期范围功能

### 👤 我的页面 (Profile)
- 个人信息管理（头像、用户名）
- 表情头像功能，支持表情选择和情绪分类
- 数据导出和导入（JSON和ZIP格式）
- WebDAV备份和同步功能
- 缓存清理功能
- 技术支持页面

### 📚 文档页面 (Docs)
- 详细的使用说明和功能介绍
- 快速导航栏，支持滚动固定
- 分类展示各项功能的使用指南

## 技术特性

### 离线使用
- 所有数据通过 `wx.setStorageSync` / `wx.getStorageSync` 存储在本地
- 无需网络连接即可使用全部功能

### 免登录使用
- 登录状态检查为可选功能
- 即使未登录也可使用所有核心功能
- 登录功能为框架预留，暂未完整实现

### 数据同步
- WebDAV备份和同步功能，支持多设备数据同步
- 增量备份功能，只备份发生变化的数据，提高备份效率
- 图片增量备份，只上传新添加或修改的图片，节省网络流量

### 数据安全
- 支持ZIP压缩备份功能，包含图片和数据的完整备份
- 备份索引文件生成和使用功能，确保备份恢复的路径一致性
- 文件校验功能，确保备份恢复的完整性

## 使用说明

### 启动方式
1. 直接在微信开发者工具中打开小程序
2. 可跳过登录步骤直接使用所有功能
3. 所有数据将保存在当前设备中

### 注意事项
- 清除小程序缓存会导致数据丢失
- 建议定期导出重要数据作为备份
- 通过WebDAV功能可以实现多设备数据同步

## 数据管理

### 数据导出
- JSON格式：完整的数据结构，适合备份和迁移
- ZIP格式：包含图片和数据的完整备份
- 导出内容包含：
  - 自定义班次模板数据（包括排列顺序）
  - 排班数据
  - 统计数据（根据排班实时计算）
  - 图片数据（仅ZIP格式）

### 数据导入
- 支持从JSON和ZIP文件导入数据
- 导入时会将各个数据恢复至对应板块：
  - 自定义班次模板板块
  - 排班板块
  - 图片数据（仅ZIP格式）
- 统计数据会根据导入的排班数据实时更新

### WebDAV同步
- 支持配置WebDAV服务器进行数据备份和同步
- 自动管理备份文件，只保留最新版本
- 多设备同步时以最新备份为准，确保数据一致性

## 开发相关

### 项目结构
```
SYwork/
├── pages/              # 页面文件
│   ├── plan/           # 计划页面
│   ├── schedule/       # 排班页面
│   ├── statistics/     # 统计页面
│   ├── profile/        # 个人中心页面
│   │   └── support/    # 技术支持页面
│   └── docs/           # 文档页面
├── utils/              # 工具函数
├── i18n/               # 国际化配置
├── docs/               # 文档说明
└── 配置文件
```

### API接口
- 使用本地存储模拟后端服务
- 通过 `utils/api.js` 封装数据操作方法
- WebDAV API集成，支持远程备份和同步

### 按需注入配置
项目已移除按需注入配置以保证稳定性。如需启用此功能，请参考以下文档：
1. `docs/lazy-loading-guide.md` - 基本配置指南
2. `docs/complete-lazy-loading-solution.md` - 完整解决方案
