# SYwork 工时记录系统

## 项目简介

SYwork 是一款功能强大的个人工时记录微信小程序，旨在帮助用户高效地规划、跟踪和管理工作时间。通过简洁直观的界面和丰富的功能，用户可以轻松制定工作计划、安排日常排班、统计工时数据，并进行个人数据管理和多设备同步。

## 功能特点

### 📋 计划管理
- 创建和管理每日工作任务
- 设置任务优先级和预计工时
- 查看任务完成状态
- 班次模板编辑、复制和批量删除功能
- 模板弹窗优化，支持滚动和触摸事件处理

### 📅 排班系统
- 制定周/月度工作排班表
- 可视化展示工作时间安排
- 支持底部滚动班次选择器组件
- 排班删除功能，在班次匹配时显示删除按钮
- 周视图图片管理功能，支持添加、查看和删除图片
- "回到今天"功能，快速切换到当前日期

### 📊 数据统计
- 图表化展示工时统计数据
- 分析工作效率和时间分配
- 导出统计报表（CSV格式）
- 本周和当月工时统计功能
- 自定义标准工时设置
- 快速选择日期范围功能

### 👤 个人中心
- 个人信息管理（头像、用户名）
- 表情头像功能，支持表情选择和情绪分类
- 数据导出和导入（JSON和ZIP格式）
- 云备份和同步功能
- 缓存清理功能
- 技术支持页面

### 📚 文档中心
- 详细的使用说明和功能介绍
- 快速导航栏，支持滚动固定
- 分类展示各项功能的使用指南

## 技术栈

- **前端框架**：微信小程序原生
- **数据存储**：微信本地存储
- **数据同步**：云备份
- **构建工具**：Gulp ^4.0.2
- **转译工具**：Babel
- **核心依赖**：@babel/runtime ^7.24.0
- **UI组件库**：@tencent/weui-wxss ^2.4.1

## 平台适配

- **HarmonyOS**：支持HarmonyOS平台，适配横屏竖屏和平板设备
- **响应式设计**：针对不同屏幕尺寸和设备类型进行优化

## 推荐开发库

- **网络请求**：wx.request（原生API）或 wx-axios
- **状态管理**：miniprogram-redux 或 miniprogram-mobx
- **工具库**：miniprogram-utils、dayjs（日期处理）
- **UI组件**：@tencent/weui-wxss、miniprogram-component-plus
- **图表库**：wx-charts、echarts-for-weixin
- **地图服务**：wx.getLocation（原生API）
- **媒体处理**：wx.chooseImage、wx.uploadFile（原生API）

## 快速开始

### 环境要求
- 微信开发者工具
- Node.js 12.0+

### 安装步骤
1. 克隆代码仓库
2. 安装依赖：`npm install`
3. 在微信开发者工具中打开项目
4. 编译并运行

1. **开发环境搭建**
   - 安装微信开发者工具
   - 克隆代码仓库
   - 安装依赖：`npm install`

2. **开发与调试**

4. **构建与发布**
   - 构建命令：`npm run build`

## 数据管理

### 数据导出
- **JSON格式**：完整的数据结构，适合备份和迁移
- **ZIP格式**：包含图片和数据的完整备份
- **导出内容**：
  - 自定义班次模板数据（包括排列顺序）
  - 排班数据
  - 统计数据（根据排班实时计算）
  - 图片数据（仅ZIP格式）

### 数据导入
- 支持从JSON和ZIP文件导入数据
- 导入时会将各个数据恢复至对应板块
- 统计数据会根据导入的排班数据实时更新

### 云备份
- 支持云备份和同步功能，实现多设备数据同步
- 自动管理备份文件，只保留最新版本
- 多设备同步时以最新备份为准，确保数据一致性

## API接口

- 使用本地存储模拟后端服务
- 通过 `utils/api.js` 封装数据操作方法
- WebDAV API集成，支持远程备份和同步

## 注意事项

- 清除小程序缓存会导致数据丢失
- 建议定期导出重要数据作为备份
- 通过云备份功能可以实现多设备数据同步
