# 数据管理API文档

## 概述

本文档详细说明了SYwork工时记录系统中数据导出和导入功能的具体规定和实现方式。

## 数据导出功能

### 功能描述
数据导出功能允许用户将系统中的所有重要数据整合成一个JSON文件，便于备份或迁移。

### 导出数据结构

```json
{
  "shiftTemplates": [
    {
      "name": "班次名称",
      "startTime": "开始时间",
      "endTime": "结束时间",
      "workHours": 工时数,
      "type": "班次类型",
      "color": "颜色代码"
    }
  ],
  "shifts": {
    "日期1": {
      "name": "班次名称",
      "startTime": "开始时间",
      "endTime": "结束时间",
      "workHours": 工时数,
      "type": "班次类型",
      "color": "颜色代码"
    }
  },
  "statistics": {
    "totalHours": "总工时",
    "totalDays": "排班天数",
    "workDays": "工作班次",
    "offDays": "休息日"
  }
}
```

### 导出内容说明

1. **自定义班次模板数据（shiftTemplates）**
   - 包含用户创建的所有班次模板
   - 保留班次模板的排列顺序
   - 包含班次名称、时间、工时、类型和颜色等完整信息

2. **排班数据（shifts）**
   - 包含用户所有的排班记录
   - 以日期为键，班次信息为值的键值对结构
   - 包含每日排班的完整信息

3. **统计数据（statistics）**
   - 根据排班数据实时计算生成
   - 包含总工时、排班天数、工作班次和休息日等统计信息

## 数据导入功能

### 功能描述
数据导入功能允许用户从JSON文件中恢复数据，将各个数据恢复至对应的板块。

### 导入数据处理流程

1. **数据验证**
   - 验证JSON文件格式是否正确
   - 检查必需的数据结构是否存在

2. **数据恢复**
   - 将班次模板数据恢复至自定义班次模板板块
   - 将排班数据恢复至排班板块

3. **数据刷新**
   - 刷新所有相关页面的数据展示
   - 重新计算并更新统计数据

### 导入后数据处理

1. **自定义班次模板板块**
   - 恢复所有班次模板
   - 保持原有排列顺序
   - 更新计划页面的班次模板列表

2. **排班板块**
   - 恢复所有排班数据
   - 更新排班页面的排班展示
   - 重新生成周/月视图

3. **统计数据**
   - 根据导入的排班数据实时更新
   - 重新计算总工时、排班天数等统计信息
   - 更新统计页面的图表展示

## 注意事项

1. **数据完整性**
   - 导出的JSON文件包含所有必要数据
   - 导入时会验证数据完整性

2. **数据同步**
   - 导入数据后会自动刷新所有相关页面
   - 统计数据会根据排班数据实时更新

3. **兼容性**
   - 导出的JSON文件可在不同设备间使用
   - 导入功能兼容历史版本的数据格式

## 使用示例

### 导出数据示例

1. 进入"我的"页面
2. 点击"导出数据"按钮
3. 系统生成JSON文件
4. 点击"分享给好友"将文件发送给他人

### 导入数据示例

1. 进入"我的"页面
2. 点击"导入数据"按钮
3. 选择之前导出的JSON文件
4. 系统自动解析并恢复数据
5. 所有页面数据自动刷新
